# Checklist - Definition of Done Oficial

## Objetivo
Definir o minimo que uma entrega deve cumprir para ser considerada pronta no projeto.

## Definition of Done
- [ ] O caso de uso ou comportamento alterado esta documentado no nivel necessario
- [ ] Se houve mudanca de fronteira, contrato ou decisao estrutural, a documentacao arquitetural foi atualizada
- [ ] Testes relevantes foram adicionados ou ajustados
- [ ] Erros esperados continuam mapeados de forma estavel
- [ ] Tenant, papel e escopo foram considerados quando aplicavel
- [ ] Auditoria e observabilidade minima foram consideradas
- [ ] Nenhuma regra de negocio nuclear foi empurrada para a camada errada
- [ ] Se houver leitura analitica, o status do dado (`partial` ou `official`) foi considerado
- [ ] Se houver API, payloads, codigos de erro e autorizacao foram considerados
- [ ] O impacto em relatorios, boletins ou dashboard foi avaliado quando aplicavel

## Regra Pratica
Se um item nao passa por este checklist, ele ainda nao esta realmente pronto, mesmo que o codigo compile ou os testes passem.

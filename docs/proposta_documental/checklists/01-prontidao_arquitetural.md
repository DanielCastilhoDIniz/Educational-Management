# Checklist - Prontidao Arquitetural

Use antes de iniciar uma nova frente importante do modulo.

- [ ] O caso de uso esta nomeado de forma explicita
- [ ] Esta claro o que e regra de dominio versus politica externa
- [ ] Existe contrato de entrada e saida da Application
- [ ] Os erros esperados estao listados
- [ ] A persistencia necessaria esta clara
- [ ] Eventos e auditoria foram considerados
- [ ] A observabilidade minima foi pensada
- [ ] Ha estrategia de teste por camada
